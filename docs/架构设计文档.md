## 2. 核心组件设计

### 2.1 Agent 基类设计

所有 Agent 实现 `NodeAction<State>` 接口：

```java
public interface NodeAction<State> {
    Map<String, Object> apply(State state) throws Exception;
}
```

Agent 生命周期：

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│  接收状态    │ ──▶ │   执行逻辑   │ ──▶ │  返回更新    │
│  (输入)     │     │  (处理)     │     │  (输出)     │
└─────────────┘     └─────────────┘     └─────────────┘
```

### 2.2 状态管理设计

状态在各个 Agent 之间流转：

```
┌─────────────────────────────────────────────────────────────┐
│                    PlanExecuteState                          │
├─────────────────────────────────────────────────────────────┤
│ 输入字段:                                                    │
│   - userQuery: String           用户问题                     │
│   - maxReplanAttempts: int      最大重规划次数               │
├─────────────────────────────────────────────────────────────┤
│ 计划字段:                                                    │
│   - plan: List<PlanStep>        执行计划                     │
│   - validationResult: String    验证结果                     │
│   - planFeasible: boolean       计划是否可行                 │
│   - replanCount: int            当前重规划次数               │
├─────────────────────────────────────────────────────────────┤
│ 执行字段:                                                    │
│   - executionResults: List      执行结果列表                 │
│   - currentStepIndex: int       当前步骤索引                 │
├─────────────────────────────────────────────────────────────┤
│ 输出字段:                                                    │
│   - finalAnswer: String         最终答案                     │
│   - completed: boolean          是否完成                     │
│   - errorMessage: String        错误信息                     │
└─────────────────────────────────────────────────────────────┘
```

### 2.3 条件边设计

条件边决定工作流的分支走向：

```java
public class PlanValidationEdge implements EdgeAction<PlanExecuteState> {

    @Override
    public Optional<String> apply(PlanExecuteState state) {
        if (state.isPlanFeasible()) {
            return Optional.of("execute");      // 跳转执行
        }

        if (state.getReplanCount() < state.getMaxReplanAttempts()) {
            return Optional.of("replan");       // 跳转重规划
        }

        return Optional.of("end");              // 终止流程
    }
}
```

## 3. 工作流编排

### 3.1 StateGraph 构建

```java
@Component
public class PlanExecuteWorkflowConfig {

    @Bean
    public CompiledWorkflow<PlanExecuteState> workflow(
            PlanAgent planAgent,
            ValidateAgent validateAgent,
            ExecuteAgent executeAgent,
            ReplanAgent replanAgent,
            PlanValidationEdge validationEdge) {

        return new StateGraph<>(PlanExecuteState::new)
            // 添加节点
            .addNode("plan", planAgent)
            .addNode("validate", validateAgent)
            .addNode("execute", executeAgent)
            .addNode("replan", replanAgent)

            // 设置入口
            .setEntryPoint("plan")

            // 添加边
            .addEdge("plan", "validate")
            .addConditionalEdges("validate", validationEdge,
                Map.of("execute", "execute", "replan", "replan", "end", END))
            .addEdge("execute", END)
            .addEdge("replan", "validate")

            .compile();
    }
}
```

```java
public ChatModel getChatModel(String modelName) {
    return modelCache.computeIfAbsent(modelName, name -> {
        ModelConfigEntity config = modelConfigRepository
            .findByModelName(name)
            .orElseThrow(() -> new RuntimeException("Model not found"));

        return createChatModel(config);
    });
}

private ChatModel createChatModel(ModelConfigEntity config) {
    return switch (config.getProvider().toLowerCase()) {
        case "ollama" -> createOllamaModel(config);
        case "openai" -> createOpenAIModel(config);
        case "deepseek" -> createDeepSeekModel(config);
        default -> throw new IllegalArgumentException("Unknown provider");
    };
}
```

## 7. 错误处理与重试机制

### 7.1 重规划策略

```java
public class PlanValidationEdge implements EdgeAction<PlanExecuteState> {

    @Override
    public Optional<String> apply(PlanExecuteState state) {
        // 1. 计划可行，执行
        if (state.isPlanFeasible()) {
            return Optional.of("execute");
        }

        // 2. 未超过最大重规划次数，重规划
        if (state.getReplanCount() < state.getMaxReplanAttempts()) {
            state.setReplanCount(state.getReplanCount() + 1);
            return Optional.of("replan");
        }

        // 3. 超过最大次数，终止
        state.setErrorMessage("Exceeded maximum replan attempts");
        return Optional.of("end");
    }
}
```

### 7.2 工具执行错误处理

```java
try {
    Object result = toolRegistry.executeTool(
        step.getToolName(),
        step.getToolInput()
    );
    step.markCompleted();
} catch (ToolNotFoundException e) {
    step.markFailed();
    log.error("Tool not found: {}", step.getToolName());
} catch (ToolExecutionException e) {
    step.markFailed();
    log.error("Tool execution failed: {}", e.getMessage());
    // 可考虑重试
}
```
