# Spring AI + LangGraph4j 多智能体系统

## 项目概述

本项目是一个基于 **Spring AI**、**LangChain4j** 和 **LangGraph4j** 构建的多智能体系统，展示了如何使用这些框架实现复杂的 AI Agent 工作流。

### 核心特性

- **计划-执行架构**：智能分解用户任务，自动规划执行步骤
- **动态工具注册**：通过数据库配置动态加载工具，无需重启服务
- **多模型支持**：支持 Ollama、OpenAI、DeepSeek 等多种 LLM 提供商
- **自动重规划**：当计划验证失败时自动调整策略
- **HTTP 接口工具**：支持通过配置定义 HTTP 接口作为工具调用

## 核心模块详解

### 1. Plan-Execute Agent

这是项目的核心模块，实现了一个智能的计划-执行工作流。

#### 工作流程图

```
         ┌─────────┐
         │  START  │
         └────┬────┘
              │
              ▼
         ┌─────────┐
         │  Plan   │  ← 制定执行计划
         └────┬────┘
              │
              ▼
         ┌─────────┐
         │ Validate│  ← 验证计划可行性
         └────┬────┘
              │
              ▼
      ┌───────┴───────┐
      │  Conditional  │
      │     Edge      │
      └───────┬───────┘
              │
     ┌────────┼────────┐
     │        │        │
     ▼        ▼        ▼
  可执行    可重规划   超过限制
     │        │        │
     ▼        ▼        ▼
┌────────┐ ┌────────┐ ┌────────┐
│Execute │ │Replan  │ │  END   │
└────┬───┘ └────┬───┘ └────────┘
     │          │
     ▼          │
┌────────┐      │
│  END   │◄─────┘
└────────┘
```

#### Agent 职责

| Agent | 类 | 职责 |
|-------|------|------|
| PlanAgent | `PlanAgent.java` | 分析用户问题，生成 JSON 格式的执行计划 |
| ValidateAgent | `ValidateAgent.java` | 验证工具是否可用、参数是否完整 |
| ExecuteAgent | `ExecuteAgent.java` | 按步骤执行计划，调用工具，生成最终答案 |
| ReplanAgent | `ReplanAgent.java` | 验证失败时重新制定计划 |

#### 状态管理

```java
public class PlanExecuteState {
    // 输入
    private String userQuery;              // 用户问题

    // 计划相关
    private List<PlanStep> plan;           // 执行计划
    private String validationResult;       // 验证结果
    private boolean planFeasible;          // 计划是否可行
    private int replanCount;               // 重规划次数
    private int maxReplanAttempts;         // 最大重规划次数

    // 执行相关
    private List<ExecutionResult> executionResults;  // 执行结果

    // 输出
    private String finalAnswer;            // 最终答案
    private boolean completed;             // 是否完成
}
```

### 执行查询

**POST** `/api/agent/query`

请求体：
```json
{
  "query": "计算 25 * 4 + 10",
  "maxReplanAttempts": 3
```

## 快速开始

### 1. 创建数据库

```sql
CREATE DATABASE IF NOT EXISTS plan_execute_db 
DEFAULT CHARACTER SET utf8mb4 
DEFAULT COLLATE utf8mb4_unicode_ci;
```

### 2. 执行建表脚本

```bash
mysql -u root -p plan_execute_db < langgraph-agent-plan-execute/src/main/resources/sql/db.sql
```

### 3. 配置环境变量

复制 `.env.example` 为 `.env` 并填写配置。

### 4. 启动应用

```bash
mvn spring-boot:run -pl langgraph-agent-plan-execute
```

### 5. 测试接口

```bash
curl -X POST http://localhost:8081/api/agent/query \
  -H "Content-Type: application/json" \
  -d '{"query": "北京今天天气怎么样？"}'
```

## 扩展开发

### 添加新工具

1. 创建工具类：

```java
@Component
public class MyNewTool {

    @Tool("执行自定义功能")
    public String execute(@P("输入参数") String input) {
        return "处理结果: " + input;
    }
}
```

2. 在数据库中注册：

```sql
INSERT INTO tool_config (tool_name, description, class_name, method_name, enabled)
VALUES ('myNewTool', '自定义工具描述', 'com.zj.ai.langgraph4j.tool.MyNewTool', 'execute', true);
```

### 添加新模型

```sql
INSERT INTO model_config (model_name, provider, base_url, model_id, is_default, enabled)
VALUES ('gpt4', 'openai', 'https://api.openai.com/v1', 'gpt-4', false, true);
```

## 设计模式

本项目采用了以下设计模式：

1. **状态图模式**：使用 LangGraph4j 的 StateGraph 定义工作流
2. **策略模式**：不同 Agent 实现相同的 NodeAction 接口
3. **工厂模式**：AgentStateFactory 创建状态实例
4. **注册表模式**：ToolRegistryService 管理工具
5. **依赖注入**：Spring 管理所有组件的生命周期
