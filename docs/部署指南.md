# 部署指南

## 1. 环境要求

### 1.1 硬件要求

| 配置项 | 最低配置 | 推荐配置 |
|--------|----------|----------|
| CPU | 2核 | 4核+ |
| 内存 | 4GB | 8GB+ |
| 磁盘 | 20GB | 50GB+ |

### 1.2 软件要求

| 软件 | 版本 |
|------|------|
| JDK | 17+ |
| MySQL | 8.0+ |
| Docker | 20.10+ (可选) |
| Docker Compose | 2.0+ (可选) |

---

## 2. 数据库准备

### 2.1 创建数据库

```sql
CREATE DATABASE plan_execute_db 
CHARACTER SET utf8mb4 
COLLATE utf8mb4_unicode_ci;
```

### 2.2 创建用户

```sql
CREATE USER 'plan_agent'@'%' IDENTIFIED BY 'your_password';
GRANT ALL PRIVILEGES ON plan_execute_db.* TO 'plan_agent'@'%';
FLUSH PRIVILEGES;
```

### 2.3 初始化表结构

应用启动时会自动创建表结构（JPA ddl-auto: update）。

---

## 3. 配置说明

### 3.1 application.yml 配置

```yaml
server:
  port: 8081

spring:
  datasource:
    url: jdbc:mysql://localhost:3306/plan_execute_db
    username: plan_agent
    password: ${DB_PASSWORD}
    
  jpa:
    hibernate:
      ddl-auto: update

# 日志级别
logging:
  level:
    com.zj.ai.langgraph4j: INFO
```

### 3.2 环境变量

| 变量名 | 说明 | 示例 |
|--------|------|------|
| CHAT_BASE_URL | LLM API 地址 | http://localhost:11434/v1 |
| CHAT_MODEL | 模型名称 | gemma4:e4b |
| CHAT_API_KEY | API 密钥 | sk-xxx |

## 5. Docker 部署

### 5.1 创建 Dockerfile

```dockerfile
FROM eclipse-temurin:17-jre-alpine

WORKDIR /app

COPY langgraph-agent-plan-execute/target/*.jar app.jar

EXPOSE 8081

ENTRYPOINT ["java", "-jar", "app.jar"]
```

### 5.2 构建镜像

```bash
mvn clean package -DskipTests
docker build -t plan-execute-agent:latest .
```

### 5.3 运行容器

```bash
docker run -d \
  --name plan-agent \
  -p 8081:8081 \
  -e CHAT_BASE_URL=http://host.docker.internal:11434/v1 \
  -e CHAT_MODEL=gemma4:e4b \
  -e CHAT_API_KEY=your-key \
  plan-execute-agent:latest
```

## 6. Docker Compose 部署

### 6.1 docker-compose.yml

```yaml
version: '3.8'

services:
  app:
    image: plan-execute-agent:latest
    ports:
      - "8081:8081"
    environment:
      - SPRING_DATASOURCE_URL=jdbc:mysql://db:3306/plan_execute_db
      - SPRING_DATASOURCE_USERNAME=plan_agent
      - SPRING_DATASOURCE_PASSWORD=your_password
      - CHAT_BASE_URL=http://ollama:11434/v1
      - CHAT_MODEL=gemma4:e4b
    depends_on:
      - db
    restart: unless-stopped

  db:
    image: mysql:8.0
    environment:
      - MYSQL_ROOT_PASSWORD=root_password
      - MYSQL_DATABASE=plan_execute_db
      - MYSQL_USER=plan_agent
      - MYSQL_PASSWORD=your_password
    volumes:
      - mysql_data:/var/lib/mysql
    restart: unless-stopped

volumes:
  mysql_data:
```

### 6.2 启动服务

```bash
docker-compose up -d
```

### 6.3 查看日志

```bash
docker-compose logs -f app
```

## 7. Kubernetes 部署

### 7.1 ConfigMap

```yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: plan-agent-config
data:
  CHAT_MODEL: "gemma4:e4b"
  CHAT_BASE_URL: "http://ollama-service:11434/v1"
```

### 7.2 Secret

```yaml
apiVersion: v1
kind: Secret
metadata:
  name: plan-agent-secret
type: Opaque
stringData:
  DB_PASSWORD: "your_password"
  CHAT_API_KEY: "your-api-key"
```

### 7.3 Deployment

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: plan-agent
spec:
  replicas: 2
  selector:
    matchLabels:
      app: plan-agent
  template:
    metadata:
      labels:
        app: plan-agent
    spec:
      containers:
      - name: app
        image: plan-execute-agent:latest
        ports:
        - containerPort: 8081
        envFrom:
        - configMapRef:
            name: plan-agent-config
        - secretRef:
            name: plan-agent-secret
        resources:
          requests:
            memory: "512Mi"
            cpu: "250m"
          limits:
            memory: "1Gi"
            cpu: "500m"
        livenessProbe:
          httpGet:
            path: /api/agent/health
            port: 8081
          initialDelaySeconds: 30
          periodSeconds: 10
        readinessProbe:
          httpGet:
            path: /api/agent/health
            port: 8081
          initialDelaySeconds: 5
          periodSeconds: 5
```

### 7.4 Service

```yaml
apiVersion: v1
kind: Service
metadata:
  name: plan-agent-service
spec:
  selector:
    app: plan-agent
  ports:
  - port: 80
    targetPort: 8081
  type: ClusterIP
```

## 9. 故障排查

### 9.1 常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 启动失败 | 数据库连接失败 | 检查数据库配置和网络 |
| 工具执行失败 | 工具类找不到 | 检查工具配置的 className |
| 模型调用失败 | API Key 无效 | 检查模型配置 |

### 9.2 日志分析

```bash
# 搜索错误日志
grep -i "error" logs/application.log

# 搜索特定异常
grep -i "Exception" logs/application.log
```
