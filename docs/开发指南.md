        // ...
        
        // 3. 更新状态
        state.setFinalAnswer("处理结果");
        
        // 4. 返回更新后的状态

## 4. 工作流配置

### 4.1 修改工作流

在 `PlanExecuteWorkflowConfig` 中修改工作流：

```java
@Bean
public StateGraph<PlanExecuteState> planExecuteStateGraph(...) {
    StateGraph<PlanExecuteState> graph = new StateGraph<>(stateFactory);

    // 添加节点
    graph.addNode("customNode", node_async(customAgent));

    // 添加边
    graph.addEdge("existingNode", "customNode");
    graph.addEdge("customNode", END);

    // 添加条件边
    graph.addConditionalEdges("nodeA", edge_async(customEdge),
            Map.of("nodeA", "nodeA", "nodeB", "nodeB", "end", END));

    return graph;
}
```

## 8. 最佳实践

### 8.1 Agent 设计原则

1. **单一职责**: 每个 Agent 只负责一个明确的任务
2. **幂等性**: 相同输入应产生相同输出
3. **容错性**: 合理处理异常情况
4. **可观测性**: 添加必要的日志

### 8.2 工具设计原则

1. **清晰的描述**: 工具描述要准确说明功能
2. **参数验证**: 验证输入参数的合法性
3. **错误处理**: 返回有意义的错误信息
4. **幂等性**: 相同参数应产生相同结果

### 8.3 代码规范

1. 使用 Lombok 简化代码
2. 遵循阿里巴巴 Java 编码规范
3. 编写清晰的注释和文档
