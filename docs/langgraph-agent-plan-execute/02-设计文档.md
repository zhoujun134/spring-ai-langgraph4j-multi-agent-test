# Plan-Execute Agent 系统实现计划

## 概述

在 `langgraph-agent-plan-execute` 模块中实现一个完整的 Plan-Execute Agent 系统，支持：
1. 动态模型加载 - 运行时切换模型，配置存储在数据库
2. 动态工具配置 - 工具定义存储在数据库，支持启用/禁用
3. Plan-Execute 工作流 - 制定计划、验证可行性、执行或重新规划

## 3. AgentState 设计

### PlanExecuteState

```java
@Getter @Setter
public class PlanExecuteState extends AgentState {
    // 输入
    private String userQuery;           // 用户原始问题

    // 计划相关

    private int currentStepIndex;       // 当前执行步骤索引
    private String validationResult;    // 验证结果

    private int maxReplanAttempts = 3;  // 最大重新规划次数

    // 执行相关

    private String currentTool;         // 当前使用的工具
    private String currentToolInput;    // 当前工具输入

    // 输出
    private String finalAnswer;         // 最终答案
    private boolean completed;          // 是否完成

    // 错误处理
    private String errorMessage;        // 错误信息

    // 构造函数
    public PlanExecuteState(Map<String, Object> data) {
        super(data);
        // 从 data 解析字段
    }

    // 转换为 Map
    public Map<String, Object> toMap() {
        return JSONUtils.convertToMap(this);
    }

    // 流式更新方法
    public PlanExecuteState withPlan(List<PlanStep> plan) {
        this.setPlan(plan);
        return this;
    }

    public PlanExecuteState withValidationResult(String result, boolean feasible) {
        this.setValidationResult(result);
        this.setPlanFeasible(feasible);
        return this;
    }
```

## 4. 工作流节点设计

### 4.1 PlanAgent - 计划制定

**职责**: 分析用户问题，生成可执行的计划步骤

```java
@Component
public class PlanAgent implements NodeAction<PlanExecuteState> {

    private final DynamicModelManager modelManager;

    @Override
    public Map<String, Object> apply(PlanExecuteState state) throws Exception {
        // 1. 获取可用工具列表
        List<ToolConfigEntity> tools = toolRegistryService.getEnabledTools();

        // 2. 构建计划生成 Prompt
        String prompt = buildPlanPrompt(state.getUserQuery(), tools);

        // 3. 调用模型生成计划
        ChatModel model = modelManager.getDefaultModel();
        String planJson = model.chat(prompt);

        // 4. 解析计划步骤
        List<PlanStep> steps = parsePlanSteps(planJson);

        // 5. 更新状态
        return state.withPlan(steps).toMap();
    }
}
```

### 4.2 ValidateAgent - 计划验证

**职责**: 验证计划的可行性

```java
@Component
public class ValidateAgent implements NodeAction<PlanExecuteState> {

    @Override
    public Map<String, Object> apply(PlanExecuteState state) throws Exception {
        // 1. 检查每个步骤的工具是否可用
        List<String> issues = new ArrayList<>();

        for (PlanStep step : state.getPlan()) {
            if (!toolRegistryService.isToolAvailable(step.getToolName())) {
                issues.add("Tool not available: " + step.getToolName());
            }
            // 检查参数有效性
            if (!validateToolInput(step)) {
                issues.add("Invalid input for tool: " + step.getToolName());
            }
        }

        // 2. 使用 LLM 验证逻辑合理性
        String validationPrompt = buildValidationPrompt(state);
        String validationResult = model.chat(validationPrompt);

        // 3. 判断是否可行
        boolean feasible = issues.isEmpty() && isPlanFeasible(validationResult);

        return state.withValidationResult(validationResult, feasible).toMap();
    }
}
```

### 4.3 ExecuteAgent - 计划执行

**职责**: 按步骤执行计划，调用工具

```java
@Component
public class ExecuteAgent implements NodeAction<PlanExecuteState> {

    @Override
    public Map<String, Object> apply(PlanExecuteState state) throws Exception {
        List<ExecutionResult> results = new ArrayList<>();

        for (PlanStep step : state.getPlan()) {
            // 1. 标记步骤为执行中
            step.setStatus(StepStatus.EXECUTING);

            // 2. 获取并执行工具
            Object result = toolRegistryService.executeTool(
                step.getToolName(),
                step.getToolInput()
            );

            // 3. 记录结果
            results.add(new ExecutionResult(step.getStepIndex(), result));
            step.setStatus(StepStatus.COMPLETED);
        }

        // 4. 生成最终答案
        String finalAnswer = generateFinalAnswer(state, results);

        state.setExecutionResults(results);
        state.setFinalAnswer(finalAnswer);
        state.setCompleted(true);

        return state.toMap();
    }
}
```

### 4.4 ReplanAgent - 重新规划

**职责**: 当计划不可行时，生成新的计划

```java
@Component
public class ReplanAgent implements NodeAction<PlanExecuteState> {

    @Override
    public Map<String, Object> apply(PlanExecuteState state) throws Exception {
        // 1. 增加重规划计数
        state.setReplanCount(state.getReplanCount() + 1);

        // 2. 构建包含失败信息的 Prompt
        String prompt = buildReplanPrompt(
            state.getUserQuery(),
            state.getPlan(),
            state.getValidationResult()
        );

        // 3. 生成新计划
        String newPlanJson = model.chat(prompt);
        List<PlanStep> newSteps = parsePlanSteps(newPlanJson);

        return state.withPlan(newSteps).toMap();
    }
}
```

## 5. 条件边设计

### PlanValidationEdge

```java
@Component
public class PlanValidationEdge implements EdgeAction<PlanExecuteState> {

    @Override
    public String apply(PlanExecuteState state) throws Exception {
        // 1. 如果计划可行，执行
        if (state.isPlanFeasible()) {
            return "execute";
        }

        // 2. 如果超过最大重规划次数，结束
        if (state.getReplanCount() >= state.getMaxReplanAttempts()) {
            state.setErrorMessage("Max replan attempts reached");
            return "end";
        }

        // 3. 否则重新规划
        return "replan";
    }
}
```

## 6. StateGraph 工作流配置

```java
@Configuration
public class PlanExecuteWorkflowConfig {

    @Bean
    public CompiledGraph<PlanExecuteState> planExecuteWorkflow(
            PlanAgent planAgent,
            ValidateAgent validateAgent,
            ExecuteAgent executeAgent,
            ReplanAgent replanAgent,
            PlanValidationEdge validationEdge,
            PlanExecuteStateFactory stateFactory) {

        // 1. 创建 StateGraph
        StateGraph<PlanExecuteState> graph = new StateGraph<>(stateFactory);

        // 2. 添加节点
        graph.addNode("plan", node_async(planAgent));
        graph.addNode("validate", node_async(validateAgent));
        graph.addNode("execute", node_async(executeAgent));
        graph.addNode("replan", node_async(replanAgent));

        // 3. 添加边
        graph.addEdge(START, "plan");
        graph.addEdge("plan", "validate");

        // 4. 添加条件边
        graph.addConditionalEdges(
            "validate",
            AsyncEdgeAction.edge_async(validationEdge),
            Map.of(
                "execute", "execute",
                "replan", "replan",
                "end", END
            )
        );

        // 5. 完成边
        graph.addEdge("execute", END);
        graph.addEdge("replan", "plan");  // 重新规划后回到计划节点

        // 6. 编译
        return graph.compile();
    }
}
```

## 7. 动态模型管理

### DynamicModelManager

```java
@Component
public class DynamicModelManager {

    private final ModelConfigRepository repository;
    private final Map<String, ChatModel> modelCache = new ConcurrentHashMap<>();

    /**
     * 获取默认模型
     */
    public ChatModel getDefaultModel() {
        ModelConfigEntity config = repository.findByIsDefaultTrue()
            .orElseThrow(() -> new RuntimeException("No default model configured"));
        return getModel(config.getModelName());
    }

    /**
     * 根据名称获取模型（懒加载 + 缓存）
     */
    public ChatModel getModel(String modelName) {
        return modelCache.computeIfAbsent(modelName, name -> {
            ModelConfigEntity config = repository.findByModelName(name)
                .orElseThrow(() -> new RuntimeException("Model not found: " + name));
            return createModel(config);
        });
    }

    /**
     * 创建模型实例
     */
    private ChatModel createModel(ModelConfigEntity config) {
        return switch (config.getProvider().toLowerCase()) {
            case "ollama" -> OllamaChatModel.builder()
                .baseUrl(config.getBaseUrl())
                .modelName(config.getModelId())
                .temperature(config.getTemperature())
                .build();
            case "openai" -> OpenAiChatModel.builder()
                .baseUrl(config.getBaseUrl())
                .apiKey(config.getApiKey())
                .modelName(config.getModelId())
                .temperature(config.getTemperature())
                .build();
            default -> throw new RuntimeException("Unknown provider: " + config.getProvider());
        };
    }

    /**
     * 刷新模型缓存
     */
    public void refreshModel(String modelName) {
        modelCache.remove(modelName);
    }
}
```

## 8. 动态工具注册

### ToolRegistryService

```java
@Service
public class ToolRegistryService {

    private final ToolConfigRepository repository;
    private final ApplicationContext applicationContext;
    private final Map<String, Object> toolInstances = new ConcurrentHashMap<>();

    /**
     * 获取所有启用的工具
     */
    public List<ToolConfigEntity> getEnabledTools() {
        return repository.findByEnabledTrueOrderByPriorityAsc();
    }

    /**
     * 检查工具是否可用
     */
    public boolean isToolAvailable(String toolName) {
        return repository.findByToolNameAndEnabledTrue(toolName).isPresent();
    }

    /**
     * 执行工具
     */
    public Object executeTool(String toolName, String input) {
        ToolConfigEntity config = repository.findByToolName(toolName)
            .orElseThrow(() -> new RuntimeException("Tool not found: " + toolName));

        Object toolInstance = getToolInstance(config);
        return invokeToolMethod(toolInstance, config.getMethodName(), input);
    }

    /**
     * 获取工具实例（反射加载）
     */
    private Object getToolInstance(ToolConfigEntity config) {
        return toolInstances.computeIfAbsent(config.getToolName(), name -> {
            try {
                Class<?> clazz = Class.forName(config.getClassName());
                return applicationContext.getAutowireCapableBeanFactory()
                    .createBean(clazz);
            } catch (Exception e) {
                throw new RuntimeException("Failed to load tool: " + name, e);
            }
        });
    }
}
```
